Let's get start with the definition of hypothesis test. (Actually it took me one hour to successfully insert math equations into my blog before I started ^_^|||)

Hypothesis test

Introduction

Instead of the basic three options of hypothesis test, i.e. \[H_0:\theta\geq\theta_0\quad v.s.\quad H_1:\theta<\theta_0\] \[H_0:\theta\leq\theta_0\quad v.s.\quad H_1:\theta>\theta_0\] \[H_0:\theta=\theta_0\quad v.s.\quad H_1:\theta\neq\theta_0\]

Let's try to write the hypothesis test in a more general way: \[H_0:\theta\in\Theta_0\quad v.s.\quad H_1:\theta\in\Theta_0^c\]

In fact, \(H_0\) and \(H_1\) doesn't need to be complementary. For instance, \(H_0:\theta\geq\theta_0\quad v.s.\quad H_1:\theta<\theta_0\) is actually equivalent to \(H_0:\theta=\theta_0\quad v.s.\quad H_1:\theta<\theta_0\). Therefore, we can write the hypothesis more generally: \[H_0:\theta\in\Theta_0\quad v.s.\quad H_1:\theta\in\Theta_1\] where \(\Theta_0\cap\Theta_1=\varnothing\)

The hypothesis testing procedure is to find a function mapping from the sample space \(\mathbb X^n\) to set \(\{0, 1\}\), i.e. \[\delta(X^n)=1\Leftrightarrow\text{reject }H_0\quad\text{and}\quad\delta(X^n)=0\Leftrightarrow\text{don't reject }H_0\] where \(X^n=(X_1,...,X_n)'\), i.e. the sample we obtain.

\(\delta(X^n)\) represents the decision about rejecting \(H_0\) or not based on the sample we got.

How to make a good decision when testing

Doing hypothesis testing is just like making a decision. The decision is that we choose to reject or not reject \(H_0\). To measure whether our decision is good or not, we need to define some measurement to quatify that. According to decision theory, we can take the value of \(\delta(X^n)\) as our action and one way of defining the risk function \(R(\theta, \delta)\) of a hypothesis test is that \[ R(\theta,\delta)=\left\{ \begin{array} PP_\theta(\delta(X^n)=1),\quad \theta\in\Theta_0\\ P_\theta(\delta(X^n)=0),\quad \theta\notin\Theta_0 \end{array}\right. \] It's easy to see that minimizing risk function is equivalent to trying best to avoid make a wrong decision.

However, no \(\delta(X^n)\) can minimize \(R(\theta, \delta)\) uniformly on \(\theta\) so that we need to make a constraint. Before it, let's introduce the definition of power and size.

Definition 1 (power function) The power function of a hypothesis test with rejection region \(R\) is the function of \(\theta\) defined by \(\beta(\theta)=P_\theta(X^n\in R)\). When \(\theta\notin\Theta_0\), \(1-\beta(\theta)\) is also known as the Type II error.

Definition 2 (size) The size of a hypothesis test is \(\alpha\) if \(\sup_{\theta\in\Theta_0}\beta(\theta)=\alpha\). The size of a test is also known as the Type I error.

It's common to control the Type I error at a specific level. Under this control, a test with a smaller Type II error, i.e. larger power will definitely be better.

Uniformly most powerful test

Then, we're wondering if there exists a best test, i.e. the test with the maximum power when its Type I error is controled. But we need to define "the best test" first, Definition 3 (Uniformly most powerful) Let \(C\) be the class of size \(\alpha\) test. Then a uniformly most powerful (UMP) test \(\delta^*_\alpha\) is s.t. \(\beta_{\delta^*_\alpha}(\theta)\geq\beta_{\delta_\alpha}(\theta)\) for \(\forall\delta_\alpha\in C\), \(\forall\theta\in\Theta_1\).

It's hard to find a UMP test only through its definition. Luckily, there is some special cases where we have some theorems about the UMP test.

Simple v.s. Simple test

Simple v.s. Simple test is the test with both \(\Theta_0\) and \(\Theta_1\) point sets. That is, \[H_0:\theta=\theta_0\quad v.s.\quad H_1:\theta=\theta_1\]

Theorem 1 (Neyman-Pearson Lemma) Consider test \(H_0:\theta=\theta_0\quad v.s.\quad H_1:\theta=\theta_1\), then the likelihood ratio test (LRT) is a unique UMP size \(\alpha\) test: \[\delta_\alpha^*(X_n)=\mathbb1\{\frac{L_n(\theta_1)}{L_n(\theta_0)}>k_\alpha\}\] where \(k_\alpha\) is s.t. \(P_{\theta_0}(\frac{L_n(\theta_1)}{L_n(\theta_0)}>k_\alpha)=\alpha\), \(L_n(\theta)\) is the likelihood function.

Simple v.s. One-sided test or One-sided v.s. One-sided test

When our test isn't a Simple v.s. Simple test but a test with the form like \[H_0:\theta=\theta_0\quad v.s.\quad H_1:\theta>\theta_1\] or \[H_0:\theta\leq\theta_0\quad v.s.\quad H_1:\theta>\theta_1\] Will the UMP test exist under certain condition? What does it look like? The answer for the first question is yes and one condition is called monotone likelihood ratio (MLR). And Karlin-Rubin theorem will help us answer the second question.

Definition 4 (Monotone likelihood ratio) A family of pdfs or pmfs has the MLR property w.r.t. statistic \(T=T(X^n)\) if the marginal pdf of \(T\), i.e. \(\{g(t|\theta):\theta\in\Theta\}\), satisfies for every \(\theta_2>\theta_1\), \(g(t|\theta_2)/g(t|\theta_1)\) is a monotone (nonincreasing or nondecreasing) function of \(t\).

Actually, any regular exponential family with \(g(t|\theta)=h(t)c(\theta)e^{w(\theta)t}\) has an MLR if \(w(\theta)\) is a nondecreasing function.

Theorem 2 (Karlin-Rubin) Consider testing \(H_0:\theta\leq(\text{or }=)\theta_0\quad v.s.\quad H_1:\theta>\theta_1\). Suppose \(T\) is a sufficient statistic for \(\theta\) and the family of pdfs or pmfs of \(T\) has an MLR. Then for any \(t_0\), the test that rejects \(H_0\) if and only if \(T>t_0\) is a UMP size \(\alpha\) test, where \(\alpha=P_{\theta_0}(T>t_0)\).

Two-sided test

The two-sided test is the test with form like \[H_0:\theta=\theta_0\quad v.s.\quad H_1:\theta\neq\theta_1\]

However, for two-sided test, no UMP test exists.

Confidence set

Before the introduction of confidence set, let's introduce the definition of set estimator first. Definition 5 (Set estimator) If \(\phi=\phi(\theta)\) is a relevant feature of \(\theta\in\Theta\), then a set estimator for \(\phi\) is a function \(C\) that maps data set \(x^n\) to the subset of \(\phi(\Theta)\).

Confidence set is a set estimator. Confidence set isn't always confidence interval because sometimes confidence estimaor \(C(x^n)\) isn't an interval. That is, confidence set is a more general terminology than confidence interval.

A specific confidence set is closely related to the confidence level we want.

Definition 6 (Confidence level) The confidence level of a set estimator \(C(x^n)\) for \(\phi(\theta)\) is \[\inf_{\theta\in\Theta}P_\theta\{C(X^n)\ni\phi(\theta)\}\] For a given \(\alpha\in[0,1]\), a set estimator \(C=C_\alpha\) is called a \(100(1-\alpha)\%\) confidence set for \(\phi(\theta)\) if its confidence level is \(\alpha\).

How to find a confidence set

Pivotal quantity

Pivotal quantity can help us construct a confidence set. The pivotal quantitiy is

Definition 7 (Pivot) A random variables \(Q(X^n,\theta)=Q(X_1,...,X_n,\theta)\) is a pivotal quantity (or pivot) if the distribution of \(Q(X^n,\theta)\) is independent of parameters \(\theta\).

Inverting a test statistic

Constructing confidence sets for a parameter \(\theta\) is equivalent to the hypothesis test for \(\theta\). The hypothesis test here doesn't need to be \(H_0:\theta=0\). In short, considering test \(H_0:\theta\in\Theta_0\), if the intersection between the confidence set \(C_\alpha\) for \(\theta\) and \(\Theta_0\) is \(\varnothing\), we can reject \(H_0\) at level \(\alpha\). I'll introduce this duality between further later.

Duality between hypothesis test and confidence set

Assume \(C_\alpha\) is a \((1-\alpha)\) level confidence set for \(\theta\). It's similar when we interested in \(\phi(\theta)\). For simplicity, we just assume \[P_\theta(\theta\in C_\alpha)=1-\alpha\] Then we can reject \(H_0:\theta=c\) if \(c\notin C_\alpha\). This makes sense because if the null hypothesis is true, the Type I error is \(\alpha\) due to \(P_\theta(\theta\notin C_\alpha)=\alpha\).

Now consider null hypothesis \(H_0:\theta=c\) where \(c\) can be different values. Then the values of \(c\) that let us fail to reject \(H_0\) at level \(\alpha\) form a confidence set of \(\theta\):

\[C_\alpha=\{c:\text{fail to reject }H_0\text{ at level }\alpha\}\] This is a confidence set at confidence level \(\alpha\) because \[P_\theta(\theta\in C_\alpha)=P(\text{fail to reject }H_0\text{ at level }\alpha)=1-\alpha\]
